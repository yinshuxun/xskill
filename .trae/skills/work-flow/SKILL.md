# Work Flow Skill

## 技能描述

Work Flow 是一个智能的工作流管理技能，帮助开发者管理工作流程，从开始新想法到提交发布的完整生命周期。

## 功能特性

- **智能交互**：输入 `work` 后自动询问你的意图，提供清晰的选项菜单
- **工作追踪**：自动记录工作状态、进度和持续时间
- **会话集成**：与 session-manager 技能集成，自动记录工作会话
- **一键发布**：支持代码提交和版本发布的一体化操作

## 使用方法

### 启动工作流管理器

```bash
npm run work
```

### 功能选项

输入 `work` 后会显示以下菜单：

1. **开始新的想法 / 新功能**
   - 选择工作类型（feature/bugfix/improvement）
   - 输入工作标题和描述
   - 自动创建会话记录
   - 保存工作状态

2. **按照 todo 继续之前的工作**
   - 显示当前工作状态
   - 查看 todo.md 中的任务
   - 更新工作进度
   - 添加新的行动项

3. **结束当前工作，记录 todo/需求文档**
   - 输入工作总结
   - 选择是否完成
   - 更新相关文档
   - 保存会话记录
   - 可选：提交代码

4. **提交发布**
   - 生成提交信息
   - 执行 git 提交
   - 推送到远程
   - 发布新版本（可选）

5. **查看当前工作状态**
   - 显示进行中的工作
   - 查看工作持续时间
   - 查看进度记录

## 配置文件

工作状态保存在 `.trae/skills/work-flow/work-state.json`，历史记录保存在 `.trae/skills/work-flow/work-history.json`。

## 集成说明

- 自动集成 `session-manager` 技能记录工作会话
- 自动读取 `tasks/todo.md` 显示当前任务
- 支持 git 提交和发布流程

## 注意事项

- 确保已配置 git 用户信息
- 确保有写入项目文件的权限
- 建议在每次工作开始前运行 `npm run work`
