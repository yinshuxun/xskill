# 会话管理最佳实践 (Session Management Best Practices)

## 1. 概述

会话管理是项目开发和团队协作中的重要环节，有效的会话管理可以确保信息的传递和决策的追踪，避免上下文丢失。本文档提供了使用 `todo.md` 和 `session-manager` 技能进行会话管理的最佳实践。

## 2. 使用 todo.md 记录会话

### 2.1 会话记录结构

在 `tasks/todo.md` 文件中，会话记录部分应包含以下内容：

- **会话标题**：清晰描述会话的主题
- **参与者**：列出所有参与会话的人员
- **时间**：会话的开始和结束时间
- **类型**：会话的类型（会议、头脑风暴、bug 修复、规划）
- **核心内容**：会话中讨论的主要内容和决策
- **行动项**：会话中分配的任务，包括负责人和截止日期
- **参考资料**：与会话相关的文档、链接等

### 2.2 会话记录模板

使用标准化的模板记录会话，确保信息的完整性和一致性：

```markdown
### [YYYY-MM-DD] 会话主题

**参与者**: [参与者姓名/角色]
**时间**: [开始时间] - [结束时间]
**类型**: #meeting / #brainstorming / #bugfix / #planning

**核心内容**:
- [要点1]
- [要点2]
- [要点3]

**行动项**:
- [ ] [任务描述] - [负责人] - [截止日期]
- [ ] [任务描述] - [负责人] - [截止日期]

**参考资料**:
- [链接1: 描述]
- [链接2: 描述]
```

### 2.3 会话记录最佳实践

- **及时记录**：会话结束后尽快记录，避免信息遗忘
- **详细但简洁**：记录关键信息，避免冗余
- **使用标签**：为不同类型的会话添加标签，便于后续查找
- **定期回顾**：定期回顾会话记录，跟踪行动项的进展
- **版本控制**：将会话记录纳入版本控制系统，确保历史记录的完整性

## 3. 使用 session-manager 技能

### 3.1 技能安装与配置

1. **确保技能可执行**：
   ```bash
   chmod +x .trae/skills/session-manager/scripts/session.js
   ```

2. **配置选项**：
   - `todoFilePath`：todo.md 文件的路径，默认为 `tasks/todo.md`
   - `sessionDir`：会话记录存储目录，默认为 `.trae/skills/session-manager/sessions`
   - `defaultParticipants`：默认参与者，默认为空数组
   - `defaultType`：默认会话类型，默认为 `meeting`

### 3.2 基本使用流程

1. **开始新会话**：
   ```bash
   session-manager start --title=项目规划会议 --type=meeting --participants=团队成员
   ```

2. **添加会话内容**：
   ```bash
   # 添加核心内容
   session-manager add --content=讨论了项目的技术架构
   
   # 添加行动项
   session-manager add --action=完成技术方案文档 --assignee=张三 --deadline=2026-03-10
   
   # 添加参考资料
   session-manager add --reference=技术架构设计文档 --url=https://example.com/docs/architecture
   ```

3. **结束会话并保存**：
   ```bash
   # 结束会话并保存到 todo.md
   session-manager end --save
   
   # 结束会话并预览内容
   session-manager end --preview
   ```

4. **管理会话记录**：
   ```bash
   # 列出所有会话
   session-manager list
   
   # 搜索会话
   session-manager search --keyword=技术架构
   
   # 查看特定会话详情
   session-manager view --id=2026-02-28-1
   ```

### 3.3 高级使用技巧

- **批量添加内容**：在会话过程中，可以多次使用 `add` 命令添加内容
- **会话分类**：根据会话类型使用不同的标签，便于后续搜索和过滤
- **参考资料管理**：添加参考资料时，包含详细的描述和链接，便于后续查阅
- **行动项跟踪**：定期使用 `list` 命令查看会话记录，跟踪行动项的进展

## 4. 集成到工作流

### 4.1 个人工作流

1. **每日站会**：使用 `session-manager` 记录每日站会的内容和行动项
2. **个人规划**：使用 `session-manager` 记录个人规划会话，跟踪个人目标的进展
3. **学习笔记**：使用 `session-manager` 记录学习会话，整理学习内容和收获

### 4.2 团队工作流

1. **项目启动会议**：使用 `session-manager` 记录项目启动会议的内容和决策
2. ** Sprint 规划**：使用 `session-manager` 记录 Sprint 规划会议的内容和任务分配
3. **回顾会议**：使用 `session-manager` 记录回顾会议的内容和改进措施
4. **技术讨论**：使用 `session-manager` 记录技术讨论的内容和决策

### 4.3 与其他工具集成

- **与项目管理工具集成**：将会话记录中的行动项同步到项目管理工具（如 Jira、Trello 等）
- **与版本控制系统集成**：将会话记录纳入版本控制系统，确保历史记录的完整性
- **与文档管理工具集成**：将会话记录中的参考资料链接到文档管理工具（如 Confluence、Notion 等）

## 5. 常见问题与解决方案

### 5.1 会话记录没有保存到 todo.md

**问题**：使用 `session-manager end --save` 命令后，会话记录没有保存到 todo.md 文件中。

**解决方案**：
- 确保 `tasks/todo.md` 文件存在且有写入权限
- 检查 `session-manager` 技能的配置是否正确
- 查看 `session-manager` 技能的日志输出，了解具体的错误信息

### 5.2 命令行参数解析错误

**问题**：使用 `session-manager` 命令时，出现参数解析错误。

**解决方案**：
- 使用正确的命令行参数格式，如 `--title=会话主题`（等号连接）
- 确保参数值不包含空格，或使用引号包围包含空格的参数值

### 5.3 会话记录内容不完整

**问题**：会话记录中的内容不完整，缺少部分信息。

**解决方案**：
- 在会话过程中及时使用 `add` 命令添加内容
- 确保 `add` 命令的参数正确，特别是 `--content`、`--action`、`--reference` 等参数
- 会话结束前，使用 `end --preview` 命令预览会话内容，确保信息完整

## 6. 最佳实践总结

1. **标准化**：使用标准化的模板记录会话，确保信息的完整性和一致性
2. **及时**：会话结束后尽快记录，避免信息遗忘
3. **详细**：记录关键信息，包括核心内容、行动项和参考资料
4. **分类**：为不同类型的会话添加标签，便于后续查找和过滤
5. **回顾**：定期回顾会话记录，跟踪行动项的进展
6. **集成**：将会话管理集成到现有的工作流中，提高工作效率
7. **版本控制**：将会话记录纳入版本控制系统，确保历史记录的完整性

通过遵循这些最佳实践，可以有效地管理会话记录，确保信息的传递和决策的追踪，避免上下文丢失，提高团队协作和项目管理的效率。
